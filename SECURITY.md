# Security Policy

## Supported versions

Security fixes are provided for the latest `0.1.x` release while that series is maintained. Older snapshots and unreleased local modifications are not supported.

## Report a vulnerability privately

Use [GitHub private vulnerability reporting](https://github.com/kilmidas/oss-maintainer-evidence/security/advisories/new). Include the affected version, impact, minimal reproduction, and a suggested mitigation when available.

Do not open a public issue for an exploitable vulnerability. Do not include access tokens, private repository content, browser data, or real credentials. A synthetic reproduction is preferred.

The maintainer will acknowledge a valid report when available, investigate it, coordinate a fix and disclosure, and credit the reporter if requested. Response times are goals rather than a service-level guarantee.

General hardening ideas without sensitive exploit details may use the public issue tracker. See [SUPPORT.md](SUPPORT.md) for non-security questions.
